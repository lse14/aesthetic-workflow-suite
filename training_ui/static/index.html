3. YAML 配置模式

可直接在这里修改 batch_size/lr/epochs 等参数，保存后再点“按YAML启动”。

每个可填参数：说明 + 推荐值

以下推荐值面向你的当前工作流（美学四维评分模型）和常见单卡训练场景。
训练配置文件（--config）
说明：训练脚本读取的主配置文件。推荐：`configs/fusion_1k_baseline.yaml`（当前默认）。
标注文件（annotations，必填）
说明：训练样本和标签来源。推荐：指向最新标注库（如 `labeling_ui/dataset/labels.db`）。
图片根目录（image_root）
说明：当标注里是相对路径时，用它定位图片。推荐：填你的图片根目录；若标注已是绝对路径可留空。
训练 split（train_split）
说明：指定训练子集名。推荐：默认留空；仅当标注含 split 字段时填 `train`。
验证 split（val_split）
说明：指定验证子集名。推荐：默认留空；仅当标注含 split 字段时填 `val`。
随机验证比例（val_ratio）
说明：仅在未提供 split 时用于切分验证集。推荐：`0.1` 到 `0.2`，常用 `0.1`。
训练轮数（epochs）
说明：完整遍历训练集次数。推荐：先用 `16`，数据更大可 `8-12`，追求上限可 `20+`。
批量大小（batch_size）
说明：每步样本数，直接影响显存。推荐：8GB 显存用 `4-8`，12GB+ 可 `8-16`。
数据加载 workers（num_workers）
说明：DataLoader 并行进程数。推荐：先用 `4`；CPU 核心多、磁盘快可试 `6-8`；过高会导致 CPU 争用。
学习率（lr）
说明：参数更新步长。推荐：`2e-4`（即 `0.0002`）作为起点；不稳定时降到 `1e-4`。
设备（device）
说明：训练运行设备。推荐：有 NVIDIA GPU 用 `cuda`；无 GPU 用 `cpu`（会慢很多）。
输出目录（output_dir）
说明：保存 checkpoint、日志和评估结果。推荐：`outputs/fusion_auto` 或按实验名单独建目录。
输出模型名（model_name）
说明：训练完成后最佳模型文件名（不含扩展名）。推荐：`best` 或按实验命名如 `fusion_aes_v1`。
模型格式（model_format）
说明：最佳模型保存格式。推荐：优先 `safetensors`（更安全，兼容当前推理 UI）；需要 PyTorch 原生可选 `pt/pth`。
评估 split（eval_split）
说明：训练后自动评估使用的数据子集。推荐：`val`（通常与验证 split 一致）。
评估 batch（eval_batch_size）
说明：只影响评估速度。推荐：先留空（跟训练 batch 一致），显存充足可比训练更大。
训练维度（target_dims）
说明：选择参与训练的评分头。推荐：初次训练全选四维；做专项微调时只勾选目标维度。
跳过自动评估（skip_eval）
说明：是否在训练结束后自动跑评估。推荐：默认“否”；批量试错时可设为“是”提速。
